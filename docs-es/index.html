    FAE Calendar — API Reference

    FAE Calendar

    Plataforma de gestión de presencias y turnos para equipos. Permite a los empleados registrar su ubicación diaria (oficina, remoto, viaje, etc.) y envía notificaciones automáticas a Microsoft Teams con el resumen del día siguiente.

    Stack

    Capa Tecnología
    Frontend React 19 + TypeScript + Vite 7
    Estilos TailwindCSS 4 + MUI
    Estado Zustand
    Backend Node.js 22 + Express + TypeScript (tsx)
    Base de datos PostgreSQL (Neon — cloud serverless)
    Notificaciones Microsoft Teams Webhooks + WhatsApp Web.js
    Internacionalización i18next (ES / EN / IT)
    Documentación TypeDoc
    Deploy Vercel (frontend + backend)

    Requisitos previos

    • Node.js 22+
    • npm 10+
    • Docker Desktop (solo para arranque con contenedores)
    • Acceso a una instancia PostgreSQL (Neon o local)

    Arranque local (sin Docker)

    1. Clonar el repositorio

    git clone <url-del-repositorio>
    cd CALENDAR

    2. Instalar dependencias de los tres niveles

    npm install
    npm install --prefix frontend
    npm install --prefix backend

    3. Configurar variables de entorno

    cp backend/.env.example backend/.env
    cp frontend/.env.example frontend/.env

    Edita backend/.env con tus credenciales reales:

    Variable Descripción
    DATABASE_URL Connection string de PostgreSQL (Neon u otro)
    JWT_SECRET String aleatorio seguro para firmar tokens
    WA_WEB_URL URL pública del frontend (para el QR de WhatsApp)
    WA_CRON_SCHEDULE Expresión cron para el envío de notificaciones
    CORS_ORIGIN Origen permitido (http://localhost:5173 en local)

    Edita frontend/.env:

    Variable Descripción
    VITE_API_URL URL base de la API (http://localhost:4000/api en local)

    4. Arrancar ambos servicios

    npm run dev
    Servicio URL
    Frontend http://localhost:5173
    Backend http://localhost:4000

    Arranque con Docker

    1. Clonar y configurar .env (mismo proceso que arriba)

    2. Construir y levantar los contenedores

    docker compose up --build

    Los servicios se arrancan en paralelo. El frontend espera a que el backend esté disponible.

    3. Primera autenticación de WhatsApp

    En el primer arranque, el backend imprimirá un código QR en los logs. Escanéalo con WhatsApp para activar el bot de notificaciones.

    docker compose logs -f backend

    La sesión persiste en el volumen wwebjs-auth entre reinicios.

    Scripts disponibles (desde la raíz)

    npm run dev            # Arranca frontend + backend en paralelo
    npm run dev:fe         # Solo frontend
    npm run dev:be         # Solo backend
    npm run build          # Build de producción del frontend
    npm run lint           # ESLint sobre el frontend
    npm run type-check     # tsc --noEmit en frontend y backend
    npm run docs           # Genera documentación TypeDoc en /docs
    npm run clean          # Elimina frontend/dist, backend/dist y /docs

    docker compose up --build        # Primera vez (construye imágenes)
    docker compose up                # Arranque normal
    docker compose down              # Para y elimina contenedores
    docker compose down -v           # Para, elimina contenedores y volúmenes
    docker compose logs -f backend   # Logs del backend en tiempo real

    Documentación de la API

    Genera la documentación TypeDoc ejecutando:

    npm run docs

    Se genera en /docs y puede abrirse con cualquier servidor estático o directamente desde el sistema de archivos.
